# Инструкция по графическому интерфейсу системы бронирования

Графический интерфейс (GUI) позволяет управлять пользователями, столами ресторана и бронированиями через окно с вкладками. Все действия выполняются через модуль `backend.py` и сохраняются в PostgreSQL.

---

## 1. Требования

- Python 3.10+
- PostgreSQL (сервер запущен)
- Настроенный файл `.env` (см. [ИНСТРУКЦИЯ.md](ИНСТРУКЦИЯ.md))
- Установленные зависимости:

```bash
pip install -r requirements.txt
```

Tkinter входит в стандартную поставку Python на macOS и большинстве дистрибутивов Linux. На Windows обычно устанавливается вместе с Python.

---

## 2. Запуск приложения

Из корня проекта:

```bash
source venv/bin/activate   # если используете виртуальное окружение
python main.py
```

Откроется окно **«Система бронирования»** размером 1100×700 пикселей.

В правом верхнем углу отображается статус подключения к базе данных, например:

```
Подключено: postgres@localhost:5432/test_db
```

Если подключение не удалось, появится диалог с описанием ошибки на русском. Проверьте файл `.env` и убедитесь, что PostgreSQL запущен.

---

## 3. Общая структура интерфейса

Приложение состоит из четырёх вкладок:

| Вкладка | Назначение |
|---------|------------|
| **База данных** | Создание таблиц в PostgreSQL |
| **Пользователи** | Управление учётными записями |
| **Столы** | Управление столами ресторана |
| **Бронирования** | Создание и редактирование броней |

На вкладках **Пользователи**, **Столы** и **Бронирования** интерфейс устроен одинаково:

- **Слева** — список записей (таблица), фильтры и кнопки поиска.
- **Справа** — форма для просмотра, создания и редактирования записи.
- **Внизу формы** — кнопки действий: Создать, Обновить, Удалить, Очистить форму.

Клик по строке в списке автоматически заполняет форму данными выбранной записи.

---

## 4. Первый запуск: порядок действий

Рекомендуемая последовательность при первом использовании:

1. Убедитесь, что PostgreSQL запущен и `.env` настроен корректно.
2. Запустите `python main.py`.
3. Перейдите на вкладку **База данных** и нажмите **Инициализировать БД**.
4. На вкладке **Пользователи** создайте хотя бы одного пользователя.
5. На вкладке **Столы** добавьте столы ресторана.
6. На вкладке **Бронирования** создайте бронирование, указав ID пользователя и ID стола из предыдущих шагов.

> Бронирование ссылается на существующих пользователей и столы. Если указать несуществующий ID, появится ошибка: *«Связанная запись не найдена»*.

---

## 5. Вкладка «База данных»

Создаёт таблицы `users`, `restaurant_tables` и `bookings`, если они ещё не существуют.

**Кнопка:** Инициализировать БД

После успешной инициализации появится сообщение *«Инициализация базы данных завершена»*, а внизу вкладки — статус о готовности таблиц.

Повторный запуск безопасен: существующие таблицы не пересоздаются.

---

## 6. Вкладка «Пользователи»

### Поля формы

| Поле | Обязательное | Описание |
|------|:------------:|----------|
| ID | — | Заполняется автоматически (только чтение) |
| Email | да | Уникальный адрес электронной почты |
| Пароль | при создании | Хранится в БД в виде SHA-256 хеша |
| ФИО | да | Полное имя пользователя |
| Телефон | нет | Контактный номер |
| Роль | да | Клиент или Администратор |
| Активен | да | Флажок активности учётной записи |

### Фильтры списка

- **Роль** — Все / Клиент / Администратор
- **Активен** — Все / Да / Нет
- **Лимит** — максимальное число записей в списке
- **Смещение** — пропуск записей (для постраничного просмотра)

### Действия

| Кнопка | Описание |
|--------|----------|
| **Обновить список** | Загрузить пользователей с учётом фильтров |
| **Найти по ID** | Найти одного пользователя и заполнить форму |
| **Создать** | Добавить нового пользователя |
| **Обновить** | Сохранить изменения выбранного пользователя |
| **Удалить** | Удалить выбранного пользователя (с подтверждением) |
| **Очистить форму** | Сбросить форму для ввода новой записи |

При обновлении пароль можно оставить пустым — тогда сохранится текущий пароль.

---

## 7. Вкладка «Столы»

### Поля формы

| Поле | Обязательное | Описание |
|------|:------------:|----------|
| ID | — | Заполняется автоматически |
| Номер | да | Уникальный номер стола (например, `T-01`) |
| Вместимость | да | Количество мест (целое число > 0) |
| Зона | да | Зал / Терраса / VIP / У окна |
| Описание | нет | Дополнительная информация о столе |
| Активен | да | Доступен ли стол для бронирования |

### Фильтры списка

- **Зона** — Все / Зал / Терраса / VIP / У окна
- **Активен** — Все / Да / Нет
- **Лимит** и **Смещение** — пагинация

### Действия

Аналогичны вкладке «Пользователи»: обновить список, найти по ID, создать, обновить, удалить, очистить форму.

---

## 8. Вкладка «Бронирования»

### Поля формы

| Поле | Обязательное | Описание |
|------|:------------:|----------|
| ID | — | Заполняется автоматически |
| ID пользователя | да | ID из вкладки «Пользователи» |
| ID стола | да | ID из вкладки «Столы» |
| Дата/время | да | Формат: `ГГГГ-ММ-ДД ЧЧ:ММ:СС` (например, `2026-06-15 19:30:00`) |
| Кол-во гостей | да | Целое число больше 0 |
| Статус | да | Ожидает / Подтверждено / Отменено / Завершено |
| Заметки | нет | Комментарий к бронированию |

### Фильтры списка

- **ID пользователя** — показать брони конкретного пользователя
- **ID стола** — показать брони конкретного стола
- **Статус** — фильтр по статусу бронирования
- **Лимит** и **Смещение** — пагинация

### Действия

Аналогичны другим вкладкам.

### Пример создания бронирования

1. На вкладке **Пользователи** запомните ID пользователя (например, `1`).
2. На вкладке **Столы** запомните ID стола (например, `2`).
3. На вкладке **Бронирования** нажмите **Очистить форму**.
4. Заполните поля:
   - ID пользователя: `1`
   - ID стола: `2`
   - Дата/время: `2026-06-20 18:00:00`
   - Кол-во гостей: `4`
   - Статус: `Ожидает`
5. Нажмите **Создать**.

## 9. Сообщения об ошибках

Все ошибки отображаются в диалоговых окнах **на русском языке**.

| Ситуация | Сообщение |
|----------|-----------|
| Дубликат email или номера стола | Запись с такими данными уже существует |
| Несуществующий ID пользователя/стола | Связанная запись не найдена |
| Не заполнены обязательные поля | Текст с указанием, что именно нужно заполнить |
| Неверный формат числа | Неверный формат числа. Введите целое число |
| Неверный формат даты | Неверный формат даты и времени |
| Проблема с подключением к БД | Описание ошибки + совет проверить `.env` |

Успешные операции подтверждаются диалогом **«Успех»** с кратким описанием (например, *«Пользователь создан»*).

## 10. Типичные сценарии

### Добавить нового клиента и забронировать стол

1. **Пользователи** → Очистить форму → заполнить Email, Пароль, ФИО → **Создать**.
2. **Столы** → убедиться, что нужный стол есть и активен.
3. **Бронирования** → указать ID клиента и стола → **Создать**.

### Изменить статус бронирования

1. **Бронирования** → кликнуть по строке в списке.
2. Изменить поле **Статус** (например, на «Подтверждено»).
3. Нажать **Обновить**.

### Деактивировать стол

1. **Столы** → выбрать стол в списке.
2. Снять флажок **Активен**.
3. Нажать **Обновить**.

### Найти все брони конкретного пользователя

1. **Бронирования** → в фильтре **ID пользователя** ввести нужный ID.
2. Нажать **Обновить список**.

## 11. Ограничения и особенности

- **Удаление пользователя или стола**, на которые ссылаются бронирования, будет отклонено базой данных (ограничение внешнего ключа). Сначала удалите или измените связанные бронирования.
- **Пароль** при создании обязателен; при редактировании — необязателен.
- **Поля с `*`** в подписи обязательны для создания новой записи.
- Приложение хранит одно постоянное соединение с БД на время работы. Закрытие окна корректно разрывает соединение.
- Список записей не обновляется автоматически — после создания или изменения данных нажимайте **Обновить список**, если запись не появилась сразу.

## 12. Структура файлов GUI

```
main.py                 — точка входа (запуск приложения)
gui/
  app.py                — главное окно и вкладки
  database_tab.py       — вкладка «База данных»
  users_tab.py          — вкладка «Пользователи»
  tables_tab.py         — вкладка «Столы»
  bookings_tab.py       — вкладка «Бронирования»
  widgets.py            — общие элементы интерфейса
  errors.py             — перевод ошибок на русский
backend.py              — бизнес-логика (используется GUI)
driver.py               — подключение к PostgreSQL
models/                 — модели User, Table, Booking
```

## 13. Устранение неполадок

### Окно не открывается

```bash
python -c "import tkinter; print('tkinter OK')"
```

Если ошибка — установите пакет `python3-tk` (Linux) или переустановите Python с поддержкой Tcl/Tk.

### Ошибка подключения к PostgreSQL

1. Проверьте, что сервер PostgreSQL запущен.
2. Убедитесь, что база из `DB_NAME` в `.env` существует.
3. Проверьте логин и пароль в `.env`.

### Таблицы не найдены

Перейдите на вкладку **База данных** и нажмите **Инициализировать БД**.

### После ошибки операции всё перестаёт работать

Перезапустите приложение (`python main.py`). В текущей версии драйвер автоматически откатывает неудачные транзакции, но перезапуск помогает при проблемах с соединением.
