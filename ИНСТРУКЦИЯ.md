# Инструкция по драйверу PostgreSQL (`db_driver`)

Модуль `db_driver` — переиспользуемый драйвер для работы с PostgreSQL через CRUD-методы. Подключение настраивается так же, как в `main.py`: переменные окружения из файла `.env`.

---

## 1. Требования

- Python 3.10+
- PostgreSQL (сервер запущен)
- Зависимости из `requirements.txt`:
  - `psycopg2-binary`
  - `dotenv`

```bash
pip install -r requirements.txt
```

---

## 2. Настройка `.env`

Скопируйте шаблон и укажите свои данные:

```bash
cp .envExample .env
```

| Переменная     | Описание              | Пример        |
|----------------|-----------------------|---------------|
| `DB_HOST`      | Хост сервера          | `localhost`   |
| `DB_PORT`      | Порт                  | `5432`        |
| `DB_NAME`      | Имя базы              | `test_db`     |
| `DB_USER`      | Пользователь          | `postgres`    |
| `DB_PASSWORD`  | Пароль                | ваш пароль    |

База из `DB_NAME` должна существовать в PostgreSQL.

---

## 3. Подключение модуля в проект

### Вариант А — проект в этой же папке

```python
from db_driver import PostgresDriver

with PostgresDriver() as db:
    users = db.read_all("users")
```

### Вариант Б — внешний проект

1. Скопируйте папку `db_driver` в свой проект.
2. Положите `.env` в корень проекта (или передайте путь к нему).
3. Установите `psycopg2-binary` и `dotenv`.

```python
from pathlib import Path
from db_driver import PostgresDriver

env_path = Path(__file__).resolve().parent / ".env"

with PostgresDriver(env_file=env_path) as db:
    user = db.create("users", {"name": "Anna", "email": "a@mail.ru"})
```

## 4. Класс `PostgresDriver`

### Создание экземпляра

```python
db = PostgresDriver()                          # .env в текущей папке
db = PostgresDriver(env_file="path/to/.env")   # явный путь к .env
```

### Контекстный менеджер (рекомендуется)

```python
with PostgresDriver() as db:
    # работа с БД
    pass
# соединение закрыто автоматически
```

### Ручное управление соединением

```python
db = PostgresDriver()
db.connect()
# ...
db.close()
```

## 5. CRUD-методы

Все методы возвращают данные как **словари** (`dict`), ключи — имена колонок.

### CREATE — вставка

**`create(table, data)`** — одна запись. Возвращает вставленную строку (`RETURNING *`), если в таблице есть `id` / serial.

```python
user = db.create("users", {
    "name": "Иван",
    "email": "ivan@mail.ru",
})
print(user["id"])
```

**`create_many(table, rows)`** — несколько записей. Возвращает количество вставленных строк.

```python
count = db.create_many("users", [
    {"name": "Анна", "email": "anna@mail.ru"},
    {"name": "Пётр", "email": "petr@mail.ru"},
])
```

---

### READ — чтение

**`read_one(table, where, columns="*")`** — одна запись по условию.

```python
user = db.read_one("users", {"email": "ivan@mail.ru"})
user = db.read_one("users", {"id": 1}, columns=["id", "name"])
```

**`read_all(table, where=None, columns="*", order_by=None, limit=None, offset=None)`** — список записей.

```python
all_users = db.read_all("users")
active = db.read_all("users", where={"name": "Иван"})
page = db.read_all("users", order_by="id", limit=10, offset=0)
```

---

### UPDATE — обновление

**`update(table, data, where)`** — возвращает число изменённых строк.

```python
updated = db.update(
    "users",
    {"name": "Иван Петров"},
    {"id": 1},
)
```

---

### DELETE — удаление

**`delete(table, where)`** — возвращает число удалённых строк.

```python
deleted = db.delete("users", {"id": 1})
```

---

### Произвольный SQL

**`execute(query, params=None, fetch=False)`** — для `CREATE TABLE`, сложных запросов и т.д.

```python
db.execute("""
    CREATE TABLE IF NOT EXISTS users (
        id SERIAL PRIMARY KEY,
        name VARCHAR(100) NOT NULL,
        email VARCHAR(100) UNIQUE NOT NULL
    )
""")

rows = db.execute("SELECT * FROM users WHERE id = %s", (1,), fetch=True)
```

## 8. Частые ошибки

| Ошибка | Решение |
|--------|---------|
| `password authentication failed` | Проверьте `DB_USER` и `DB_PASSWORD` в `.env` |
| `database "test_db" does not exist` | Создайте базу: `CREATE DATABASE test_db;` |
| `relation "users" does not exist` | Создайте таблицу (см. раздел 6) |
| Переменные не подхватываются | Сохраните `.env`, запускайте скрипт из папки проекта |
